# 銀河はぐれ団 (GINGA HAGUREDAN)

スマホ縦画面向けのブラウザゲーム。HTML / CSS / JavaScript + [Three.js](https://threejs.org/) で構築。

## 遊び方（ローカル）

静的サイトなので、適当な静的サーバーで `index.html` を配信するだけ。

```bash
npx serve .
# または
python -m http.server 4321
```

ブラウザで `http://localhost:4321` を開く（スマホ縦画面 or デベロッパーツールのモバイル表示推奨）。

## ゲームサイクル

```
タイトル → ワールドマップ → 航行フェーズ(Three.js シューティング) → 結果画面
   → 戦闘(RPG) → 勝利 → 仲間候補出現 → スカウト（成功 / 失敗 / スルー）
   → 図鑑登録 → 次の星へ
```

スカウトは **戦闘中ではなく、戦闘に勝利した後** に発生する「仲間候補イベント」です。
戦闘画面にスカウトコマンドはありません。

## ファイル構成

| ファイル | 役割 |
|----------|------|
| `index.html` | 全画面のマークアップ（タイトル/マップ/シューティング/戦闘/スカウト/図鑑） |
| `styles.css` | スマホ縦画面レイアウトと演出 |
| `app.js`     | ゲームロジック、Three.jsシーン、状態遷移、localStorage保存 |
| `assets/`    | 差し替え用PNG素材（無くても絵文字で動作） |

## 画像の差し替え

PNGが無くても絵文字フォールバックで動作する。画像を置くと自動で使われる。

| 種別 | 置き場所 | 例 |
|------|----------|----|
| 仲間キャラ | `assets/characters/<id>.png` | `robo.png`, `neko.png` |
| 敵 | `assets/enemies/<id>.png` | `slime.png`, `boss.png` |
| 飛行船 | `assets/ships/` | （現状は簡易3Dジオメトリ） |
| タイトル / UI / シューティング素材 | `assets/title/` `assets/ui/` `assets/shooting/` | |

ID一覧は `app.js` 冒頭の `ALLIES` / `ENEMIES` / `STARS` を参照。

## 航行フェーズ（Three.js シューティング）

次の星へ向かう「航行パート」。敵を倒すだけでなく、クリスタルを集め・危険を避け・
ミッションを狙い、次のRPG戦闘を有利にするのが目的です。

### 操作

- **ドラッグ位置に飛行船が追従**（タッチ / マウス）。**上下左右** すべてに動ける
- 攻撃は自動
- 制限時間 **30秒**（`STAGE_TIME`）。HPが残ったまま時間切れ＝到達成功 → 結果画面 → 戦闘へ。
  HP0で墜落＝結果画面 → ワールドマップへ戻る

### 拾えるもの（クリスタル）

| 種類 | 効果 | 出現率 |
|------|------|--------|
| 通常クリスタル（水色） | +5💎 | 高 |
| 大クリスタル（金） | +20💎 | 低 |
| 回復クリスタル（緑） | HP +1（最大3まで） | 中 |

### 危険物（隕石）

| 種類 | 効果 | 出現率 |
|------|------|--------|
| 隕石（茶） | HP -1 | 高 |
| 赤い隕石 | HP -2 | 低 |

### ミッションとボーナス

航行開始時に **1つだけ** ミッションが提示されます（`app.js` の `MISSIONS`）。
成功すると、結果画面でボーナスを獲得し **次のRPG戦闘 / スカウトに反映** されます。

| ミッション | 成功ボーナス | 反映先 |
|------------|--------------|--------|
| クリスタルを5個 集めろ | クリスタル +20 | 報酬に加算 |
| 10秒間 ノーダメージで耐えろ | 味方HP +1 | 戦闘開始時の全員HP |
| 大クリスタルを1個 拾え | スカウト成功率 +5% | スカウト判定 |
| 敵を3体 倒せ | 敵HP -10% | 戦闘開始時の敵HP |

### 結果画面

航行終了後、戦闘へ行く前に1枚はさみます。表示内容：

- 獲得クリスタル / 受けたダメージ / 倒した敵数 / 到達の成否
- ミッション成功 / 失敗
- 獲得したボーナス

「次へ」で、到達成功なら戦闘へ（ボーナス反映）、墜落ならワールドマップへ戻ります。

## 戦闘コマンド

主人公は戦わず、仲間（最大4人）が戦う。コマンドは以下の4つ:

- **たたかう** … 仲間全員が通常攻撃
- **スキル** … 仲間全員が威力アップの一撃
- **どうぐ** … 「ほしのしずく」で全員回復（クリスタル消費）
- **にげる** … 一定確率で離脱

## スカウト（戦闘勝利後）

戦闘に勝利すると、その星の **仲間候補** と出会い、専用画面に遷移します。
画面には候補の「名前 / レア度 / 種族タグ / 必要クリスタル / 成功率」が表示されます。

- **スカウトする** … シューティングで貯めたクリスタルを使い、成功率に応じた確率で判定
  - 成功 → パーティに加入（4人を超える場合は末尾と入れ替え）／クリスタル消費／図鑑に「加入済み」記録
  - 失敗 → 仲間にならない／クリスタル消費なし（MVP）
- **スルーする** … 仲間にならない／クリスタル消費なし

必要クリスタルと成功率はレア度で決まります（`app.js` の `RARITY`）。

### 図鑑の登録ルール

- **発見済み** … 戦闘勝利後に出会った時点で登録（スカウトの成否・スルーに関係なく残る）
- **加入済み** … スカウト成功でパーティ加入したときに追加記録

## セーブ

`localStorage`（キー: `ginga-haguredan-save-v1`）に自動保存。
タイトルの「データ消去」または DevTools コンソールで `GAME.reset()`。

## デバッグ用フック

開発中は DevTools コンソールから操作可能:

```js
GAME.save              // 現在のセーブ
GAME.goShooting(0)     // 0番目の星の航行フェーズへ
GAME.goBattle(0)       // 0番目の星の戦闘へ
GAME.winShooting()     // 航行を即クリア（到達）扱いに
GAME.completeMission() // 現在のミッションを成功扱いに
GAME.setBonus("enemyHp" | "allyHp" | "scoutRate" | "crystal") // 航行ボーナスを手動セット
GAME.bonus             // 現在の航行ボーナス
GAME.sh                // 航行フェーズの内部状態（ship 等）
GAME.reset()           // セーブ初期化
```
