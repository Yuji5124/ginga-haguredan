# 銀河はぐれ団 (GINGA HAGUREDAN)

スマホ縦画面向けのブラウザゲーム。HTML / CSS / JavaScript + [Three.js](https://threejs.org/) で構築。

## 遊び方（ローカル）

静的サイトなので、適当な静的サーバーで `index.html` を配信するだけ。

```bash
npx serve .
# または
python -m http.server 4321
```

ブラウザで `http://localhost:4321` を開く（スマホ縦画面 or デベロッパーツールのモバイル表示推奨）。

## ゲームサイクル

```
タイトル → ワールドマップ → 航行フェーズ(Three.js シューティング) → 結果画面
   → 戦闘(RPG) → 勝利 → 仲間候補出現 → スカウト（成功 / 失敗 / スルー）
   → 図鑑登録 → 次の星へ
```

スカウトは **戦闘中ではなく、戦闘に勝利した後** に発生する「仲間候補イベント」です。
戦闘画面にスカウトコマンドはありません。

## ステージと敵（全20ステージ）

ワールドマップは **20ステージ**（`app.js` の `RAW_STAGES`）。前のステージをクリアすると次が解放されます。
各ステージに敵が1体おり、敵の **HP / こうげき / 報酬 / スカウト出現レア度帯** は
ステージ番号から自動スケールします（終盤ほど強く、報酬も高い）。

敵は4系統に分類され、戦闘開始時に **系統＋ギミック（説明）** を提示します。

| 系統 | 内容 | 例 |
|------|------|----|
| 宇宙生物系 | モンスター・星獣 | 流星オオカミ、星喰い幼獣グラトン |
| ロボ・AI系 | 管理・機械・兵器 | カチカチ検問ロボ、運命管理AI ラックレス |
| 悪人系 | 海賊・商人・団長 | 商人ガメル、海賊キャプテン・ザバ |
| 概念系 | 運命・記憶・時間 | 記憶ぬすみノイズ、運命固定装置ラストギア |

> ギミック（アイテム奪取・スキル封印・状態異常・強化コピー・フェーズ制 等）は現状
> **戦闘導入での説明表示**にとどめた簡易版です。所持品/MP/状態異常などのシステムは
> 後続バージョンで段階的に実装予定。

**ラスボス（ステージ20）運命固定装置ラストギア**：登場時に固有セリフを表示。
「弱い者が偶然強くなる世界は不安定だから、全員の価値を最初から固定する」という思想で、
偶然集まったはぐれ者たちのテーマと対になる存在です。

## 画像の差し替え

PNGが無くても絵文字フォールバックで動作する。画像を置くと自動で使われる。

| 種別 | 置き場所 | 例 |
|------|----------|----|
| 仲間キャラ | `assets/characters/<id>.png` | `c1.png`, `c100.png`（id は `c`+No） |
| 敵 | `assets/enemies/<id>.png` | `slime.png`, `boss.png` |
| 飛行船 | `assets/ships/` | （現状は簡易3Dジオメトリ） |
| タイトル / UI / シューティング素材 | `assets/title/` `assets/ui/` `assets/shooting/` | |

ID一覧は `app.js` 冒頭の `ALLIES` / `ENEMIES` / `STARS` を参照。

## 航行フェーズ（Three.js シューティング）

次の星へ向かう「航行パート」。敵を倒すだけでなく、クリスタルを集め・危険を避け・
ミッションを狙い、次のRPG戦闘を有利にするのが目的です。

### 操作

- **ドラッグ位置に飛行船が追従**（タッチ / マウス）。**上下左右** すべてに動ける
- 攻撃は自動
- 制限時間 **30秒**（`STAGE_TIME`）。HPが残ったまま時間切れ＝到達成功 → 結果画面 → 戦闘へ。
  HP0で墜落＝結果画面 → ワールドマップへ戻る

### 航行前画面（仲間効果）

航行に出る前に「今回の航行効果」を表示し、「航行開始」で出発します。
パーティ編成（最大4人）に応じて、各仲間の航行ボーナスが重ねて適用されます（`computeNavEffects`）。
航行ボーナスは8種あり、各仲間にレア度（No）から自動割当されます（`NAV_DEFS` / `NAV_ORDER`）。

| 航行ボーナス | 効果 |
|--------------|------|
| 射撃速度アップ | 自動弾の間隔を短縮（×0.85、重ねがけ可） |
| クリスタル獲得 +10% | 拾ったクリスタルの💎を増加 |
| ミッション報酬 強化 | ミッション成功ボーナスを強化（後述） |
| 隕石ダメージ 低確率で無効 | 隕石被弾を確率(20%/人)で無効化 |
| 敵撃破で +1💎 | 撃破時のクリスタルを加算 |
| 大クリスタル +10 | 大クリスタルの価値を上乗せ |
| 開始時バリア ×1 | 最初の被弾を1回ぶん肩代わり |
| 隕石スピード -10% | 隕石・敵の接近を遅くする |

### 拾えるもの（クリスタル）

| 種類 | 効果 | 出現率 |
|------|------|--------|
| 通常クリスタル（水色） | +5💎 | 高 |
| 大クリスタル（金） | +20💎 | 低 |
| 回復クリスタル（緑） | HP +1（最大3まで） | 中 |

### 危険物（隕石）

| 種類 | 効果 | 出現率 |
|------|------|--------|
| 隕石（茶） | HP -1 | 高 |
| 赤い隕石 | HP -2 | 低 |

### ミッションとボーナス

航行開始時に **1つだけ** ミッションが提示されます（`app.js` の `MISSIONS`）。
成功すると、結果画面でボーナスを獲得し **次のRPG戦闘 / スカウトに反映** されます。

| ミッション | 成功ボーナス | 反映先 |
|------------|--------------|--------|
| クリスタルを5個 集めろ | クリスタル +20 | 報酬に加算 |
| 10秒間 ノーダメージで耐えろ | 味方HP +1 | 戦闘開始時の全員HP |
| 大クリスタルを1個 拾え | スカウト成功率 +5% | スカウト判定 |
| 敵を3体 倒せ | 敵HP -10% | 戦闘開始時の敵HP |

**ミドリ星人がパーティにいると報酬が強化**されます: クリスタル +20→**+30** /
味方HP +1→**+2** / スカウト成功率 +5%→**+8%** / 敵HP -10%→**-15%**。

### 結果画面

航行終了後、戦闘へ行く前に1枚はさみます。表示内容：

- **評価ランク S / A / B / C** ＋一言コメント
  （S「完璧な航行！」/ A「いい航行だった！」/ B「なんとか突破！」/ C「船体ボロボロ…」）
- 獲得クリスタル / 受けたダメージ / 倒した敵数 / 到達の成否
- ミッション成功 / 失敗（成功時は強調演出）
- 獲得したボーナス（強化時は「強化↑」表示）

ランクは「到達したか」「被ダメージ」「ミッション成否」から決定します。
「次へ」で、到達成功なら戦闘へ（ボーナス反映）、墜落ならワールドマップへ戻ります。

## 戦闘コマンド

主人公は戦わず、仲間（最大4人）が戦う。コマンドは以下の4つ:

- **たたかう** … 仲間全員が通常攻撃
- **スキル** … 仲間全員が威力アップの一撃
- **どうぐ** … 「ほしのしずく」で全員回復（クリスタル消費）
- **にげる** … 一定確率で離脱

テンポ・視認性の調整: 戦闘ログは **最新5件** のみ表示（`LOG_MAX`）。攻撃〜反応の待ち時間を短縮。
敵HPバーを大きくし、**現在HP / 最大HP の数値** も併記します。

## 仲間（100人ロスター）

仲間は **100人 / レア度1〜10**（各10人）です。`app.js` の `RAW_ALLIES` に
`[No, 名前, 絵文字, 見た目・設定, 得意なこと]` を持ち、そこから以下を自動算出します。

- **レア度** … No から自動（No1〜10＝★1、…、No91〜100＝★10）
- **HP / こうげき** … レア度から自動（`30 + ★×14` / `8 + ★×4`）
- **航行ボーナス** … 8種（`NAV_DEFS`）を順番に自動割当（簡易）
- **スカウトのボイス** … レア度帯ごとの口調（`voiceForRarity`）

「得意なこと（`skill`）」と「見た目・設定（`setting`）」は図鑑・パーティ画面に
**説明テキスト** として表示します（個別のゲーム効果は未実装の簡易版）。
初期メンバーは `STARTER_ID`（No.1 まいごロボ・ピノ）。

> 旧セーブ（8人時代の仲間ID）は読み込み時に自動移行され、存在しないIDは除去・初期メンバーを補完します。

## スカウト（戦闘勝利後）

戦闘に勝利すると、その星の **レア度帯** に応じた仲間候補と出会い、専用画面に遷移します。
画面には候補の「名前 / レア度 / 得意なこと / 必要クリスタル / 成功率」が表示されます。
出会える候補のレア度は星ごとに決まります（`STARS` の `rMin` / `rMax`：序盤の星ほど低レア）。

- **スカウトする** … シューティングで貯めたクリスタルを使い、成功率に応じた確率で判定
  - 成功 → パーティに加入（4人を超える場合は末尾と入れ替え）／クリスタル消費／図鑑に「加入済み」記録
  - 失敗 → 仲間にならない／クリスタル消費なし（MVP）
- **スルーする** … 仲間にならない／クリスタル消費なし

必要クリスタルと成功率はレア度で決まります（`app.js` の `RARITY`、レア度1〜10）。
スカウト成功 / 失敗 / スルーのそれぞれで、候補キャラの **一言（ボイス）** が表示されます。

### 図鑑（100体）の登録ルール

- **発見済み** … 戦闘勝利後に出会った時点で登録（スカウトの成否・スルーに関係なく残る）
- **加入済み** … スカウト成功でパーティ加入したときに追加記録
- 図鑑セルにはレア度（★N）も表示。タップ長押し等でツールチップに設定・得意を表示

## セーブ

`localStorage`（キー: `ginga-haguredan-save-v1`）に自動保存。
タイトルの「データ消去」または DevTools コンソールで `GAME.reset()`。

## デバッグ用フック

開発中は DevTools コンソールから操作可能:

```js
GAME.save              // 現在のセーブ
GAME.goShooting(0)     // 0番目の星の航行フェーズへ
GAME.goBattle(0)       // 0番目の星の戦闘へ
GAME.winShooting()     // 航行を即クリア（到達）扱いに
GAME.completeMission() // 現在のミッションを成功扱いに
GAME.setBonus("enemyHp" | "allyHp" | "scoutRate" | "crystal") // 航行ボーナスを手動セット
GAME.bonus             // 現在の航行ボーナス
GAME.sh                // 航行フェーズの内部状態（ship 等）
GAME.reset()           // セーブ初期化
```

## v0.7 アップデート

### RPG戦闘画面の宇宙風化（Three.js）

- 戦闘画面に Three.js 背景 `#battle-canvas` を追加（既存の戦闘ロジックは不変、見た目のみ強化）
- 軽量演出：星粒子 ＋ ゆっくり動く星雲風グラデ ＋ 敵の背後に「オーラ」（敵の系統で色が変化、ボスは大きめ）
- 味方カード・敵HPバー・コマンドは背景の上に重ねて表示（宇宙空間で対峙する構図）
- 関数: `initBattleScene()` / `renderBattleScene()` / `updateBattleSceneEnemy(enemy)` /
  `playBattleHitEffect()` / `playBattleSkillEffect()` / `disposeBattleScene()`
- **WebGLコンテキスト対策**：戦闘レンダラは1つだけ生成して再利用し、
  戦闘画面を離れると描画ループが自動停止（`currentScreen !== "battle"` で停止）

### ラスボス最終演出の土台（仮）

- フラグ `isFinalBossBattle` / `finalPhase` を追加
- ラスボス（ステージ20）のHPが0になると、通常勝利ではなく **最終フェーズ（仮）** に入る
- 仮の最終演出画面：残ったコマンドは「スカウト」だけ。押すたびに表示が崩れていく
  （スカウト → スカウト → スカウ… → ス… → 魔王の最後のセリフ）
- 魔王の最後のセリフ：「そんな未来を知っていれば、私にも…」→ エンディング仮画面へ
- ※ 仲間全滅や飛行船が庇う本演出は次フェーズ。今回はコード構造のみ
