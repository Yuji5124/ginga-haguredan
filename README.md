# 銀河はぐれ団 (GINGA HAGUREDAN)

スマホ縦画面向けのブラウザゲーム。HTML / CSS / JavaScript + [Three.js](https://threejs.org/) で構築。

## 遊び方（ローカル）

静的サイトなので、適当な静的サーバーで `index.html` を配信するだけ。

```bash
npx serve .
# または
python -m http.server 4321
```

ブラウザで `http://localhost:4321` を開く（スマホ縦画面 or デベロッパーツールのモバイル表示推奨）。

## ゲームサイクル

```
タイトル → ワールドマップ → 航行フェーズ(Three.js シューティング) → 結果画面
   → 戦闘(RPG) → 勝利 → 仲間候補出現 → スカウト（成功 / 失敗 / スルー）
   → 図鑑登録 → 次の星へ
```

スカウトは **戦闘中ではなく、戦闘に勝利した後** に発生する「仲間候補イベント」です。
戦闘画面にスカウトコマンドはありません。

## ステージと敵（全20ステージ）

ワールドマップは **20ステージ**（`app.js` の `RAW_STAGES`）。前のステージをクリアすると次が解放されます。
各ステージに敵が1体おり、敵の **HP / こうげき / 報酬 / スカウト出現レア度帯** は
ステージ番号から自動スケールします（終盤ほど強く、報酬は控えめに上昇）。
各ステージは **1回勝利すればクリア** です。クリア済みステージにも再挑戦でき、クリスタル稼ぎ・スカウト・図鑑登録に使えます。

敵は4系統に分類され、戦闘開始時に **系統＋ギミック（説明）** を提示します。

| 系統 | 内容 | 例 |
|------|------|----|
| 宇宙生物系 | モンスター・星獣 | 流星オオカミ、星喰い幼獣グラトン |
| ロボ・AI系 | 管理・機械・兵器 | カチカチ検問ロボ、運命管理AI ラックレス |
| 悪人系 | 海賊・商人・団長 | 商人ガメル、海賊キャプテン・ザバ |
| 概念系 | 運命・記憶・時間 | 記憶ぬすみノイズ、運命固定装置ラストギア |

> ギミック（アイテム奪取・スキル封印・状態異常・強化コピー・フェーズ制 等）は現状
> **戦闘導入での説明表示**にとどめた簡易版です。ショップ所持品・スキル・パッシブは v0.8 で実装済み。

**ラスボス（ステージ20）運命固定装置ラストギア**：登場時に固有セリフを表示。
「弱い者が偶然強くなる世界は不安定だから、全員の価値を最初から固定する」という思想で、
偶然集まったはぐれ者たちのテーマと対になる存在です。

## 航行フェーズ（Three.js シューティング）

次の星へ向かう「航行パート」。敵を倒すだけでなく、クリスタルを集め・危険を避け・
ミッションを狙い、次のRPG戦闘を有利にするのが目的です。

### 操作

- **ドラッグ位置に飛行船が追従**（タッチ / マウス）。画面下で左右移動
- 攻撃は自動
- 制限時間 **24秒**（`STAGE_TIME`）。HPが残ったまま時間切れ＝到達成功 → 結果画面 → 戦闘へ。
  HP0で墜落＝結果画面 → ワールドマップへ戻る

### 航行前画面（仲間効果）

航行に出る前に「今回の航行効果」を表示し、「航行開始」で出発します。
パーティ編成（最大4人）に応じて、各仲間の航行ボーナスが重ねて適用されます（`computeNavEffects`）。
航行ボーナスは8種あり、各仲間にレア度（No）から自動割当されます（`NAV_DEFS` / `NAV_ORDER`）。

| 航行ボーナス | 効果 |
|--------------|------|
| 射撃速度アップ | 自動弾の間隔を短縮（×0.85、重ねがけ可） |
| クリスタル獲得 +10% | 拾ったクリスタルの💎を増加 |
| ミッション報酬 強化 | ミッション成功ボーナスを強化（後述） |
| 隕石ダメージ 低確率で無効 | 隕石被弾を確率(20%/人)で無効化 |
| 敵撃破で +1💎 | 撃破時のクリスタルを加算 |
| 大クリスタル +2 | 大クリスタルの価値を上乗せ |
| 開始時バリア ×1 | 最初の被弾を1回ぶん肩代わり |
| 隕石スピード -10% | 隕石・敵の接近を遅くする |

### 拾えるもの（クリスタル）

| 種類 | 効果 | 出現率 |
|------|------|--------|
| 通常クリスタル（水色） | +1〜2💎 | 中 |
| 大クリスタル（金） | +8〜10💎 | かなり低 |
| 回復クリスタル（緑） | HP +1 | 低 |

### 危険物（隕石）

| 種類 | 効果 | 出現率 |
|------|------|--------|
| 隕石（茶） | HP -1 | 高 |
| 赤い隕石 | HP -2 | 低 |

### ミッションとボーナス

航行開始時に **1つだけ** ミッションが提示されます（`app.js` の `MISSIONS`）。
成功すると、結果画面でボーナスを獲得し **次のRPG戦闘 / スカウトに反映** されます。

| ミッション | 成功ボーナス | 反映先 |
|------------|--------------|--------|
| クリスタルを5個 集めろ | クリスタル +12 | 報酬に加算 |
| 10秒間 ノーダメージで耐えろ | 味方HP +1 | 戦闘開始時の全員HP |
| 大クリスタルを1個 拾え | スカウト成功率 +5% | スカウト判定 |
| 敵を3体 倒せ | 敵HP -10% | 戦闘開始時の敵HP |

**ミッション報酬 強化の航行効果があると報酬が強化**されます: クリスタル +12→**+18** /
味方HP +1→**+2** / スカウト成功率 +5%→**+8%** / 敵HP -10%→**-15%**。

### 結果画面

航行終了後、戦闘へ行く前に1枚はさみます。表示内容：

- **評価ランク S / A / B / C** ＋一言コメント
  （S「完璧な航行！」/ A「いい航行だった！」/ B「なんとか突破！」/ C「船体ボロボロ…」）
- 獲得クリスタル / 受けたダメージ / 倒した敵数 / 到達の成否
- ミッション成功 / 失敗（成功時は強調演出）
- 獲得したボーナス（強化時は「強化↑」表示）

ランクは「到達したか」「被ダメージ」「ミッション成否」から決定します。
「次へ」で、到達成功なら戦闘へ（ボーナス反映）、墜落ならワールドマップへ戻ります。

## 戦闘コマンド

主人公は戦わず、仲間（最大4人）が戦う。コマンドは以下の4つ:

- **たたかう** … 仲間全員が通常攻撃
- **スキル** … 生存している仲間が、それぞれの固有スキルを1戦1回使う
- **どうぐ** … ショップで買った戦闘アイテムを使う
- **にげる** … 一定確率で離脱

スキルは `SKILLS` の8種（単体攻撃 / 全体攻撃 / 回復 / 防御アップ / 敵攻撃ダウン /
クリティカル / 次攻撃強化 / 低確率大ダメージ）を、仲間Noから自動割当します。
パッシブは `PASSIVES` の5種（防御+2 / 攻撃+3 / 素早さ+4 / 最大HP+10 / 会心率+12%）を戦闘開始時に反映します。

## タグシナジー

仲間はそれぞれ2〜3個のタグ（機械 / 動物 / 精霊 / 宇宙人 / はぐれもの / 回復 / 攻撃 / 守り / 不思議 / 低レアなど）を持ちます。
パーティ4人のタグを集計し、条件を満たすと小さな組み合わせ効果が発動します。

| シナジー | 条件 | 効果 |
|----------|------|------|
| 機械リンク | 機械 2体以上 | 防御 +10% |
| 動物の勘 | 動物 2体以上 | 回避率 +8% |
| 星霊の余韻 | 精霊 2体以上 | 戦闘終了後、味方を少し回復 |
| 宇宙人ネット | 宇宙人 2体以上 | シューティングのクリスタル獲得 +10% |
| はぐれものの絆 | はぐれもの 2体以上 | スカウト成功率 +5% |
| 回復役の見守り | 回復 1体以上 | 毎ターン終了時、低HPの味方を少し回復 |
| 攻撃陣形 | 攻撃 2体以上 | 通常攻撃 +8% |
| 守りの構え | 守り 2体以上 | 戦闘開始時にバリア1回 |
| 不思議な追い風 | 不思議 2体以上 | ミッション成功ボーナスを強化 |
| 小さな意地 | 低レア 2体以上 | 低レアキャラのHP +10% |

発動中シナジーはワールドマップ、パーティ画面、航行前画面、戦闘開始ログに表示されます。

テンポ・視認性の調整: 戦闘ログは **最新5件** のみ表示（`LOG_MAX`）。攻撃〜反応の待ち時間を短縮。
敵HPバーを大きくし、**現在HP / 最大HP の数値** も併記します。

| アイテム | 価格 | 効果 |
|----------|------|------|
| 小回復カプセル | 10💎 | 戦闘中、HPが少ない味方1人を20回復 |

| バリアオーブ | 30💎 | 戦闘中、次に受けるダメージを1軽減 |
| スカウトビーコン | 40💎 | 次のスカウト挑戦の成功率 +10%（挑戦時に自動消費） |

## 仲間（60人ロスター）

仲間は **60人 / レア度1・2・4・6・8・10**（各10人）です。正式IDは `c1`〜`c60`。`app.js` の `CHARACTER_CATALOG` に
`{ id, name, rarity, role }` を持ち、そこから以下を自動算出します。
現在のPNG画像実装対象は `c1`〜`c60` です。PNGが無い場合のみ絵文字フォールバックで動作します。

- **レア度** … カタログ値を使用（c1〜c10＝★1、c11〜c20＝★2、c21〜c30＝★4、c31〜c40＝★6、c41〜c50＝★8、c51〜c60＝★10）
- **ロール** … `role` を使用し、表示名・タグ・説明文を補完
- **HP / こうげき / 防御 / 素早さ / レベル** … レア度から自動
- **戦闘スキル / パッシブ** … No から自動割当
- **航行ボーナス** … 8種（`NAV_DEFS`）を順番に自動割当（簡易）
- **タグ** … ロール・説明・戦闘スキルから2〜3個を付与
- **スカウトのボイス** … レア度帯ごとの口調（`voiceForRarity`）

「得意なこと（`skill`）」と「見た目・設定（`setting`）」は図鑑・パーティ画面に
**説明テキスト** として表示します。戦闘で使うスキル・パッシブとは別枠です。
初期メンバーは `STARTER_ID`（No.1 ロボ太）。

> 旧セーブ（8人時代や旧ロスターの仲間ID）は読み込み時に自動移行され、存在しないIDは除去・初期メンバーを補完します。

## スカウト（戦闘勝利後）

戦闘に勝利すると、その星の **レア度帯** に応じた仲間候補と出会い、専用画面に遷移します。
画面には候補の「名前 / レア度 / 得意なこと / 必要クリスタル / 成功率」が表示されます。
出会える候補のレア度は星ごとに決まります（`STARS` の `rMin` / `rMax`：序盤の星ほど低レア）。

- **スカウトする** … シューティングで貯めたクリスタルを使い、成功率に応じた確率で判定
  - 成功 → パーティに加入（4人を超える場合は末尾と入れ替え）／クリスタル消費／図鑑に「加入済み」記録
  - 失敗 → 仲間にならない／クリスタル消費なし（MVP）
- **スルーする** … 仲間にならない／クリスタル消費なし

必要クリスタルと成功率はレア度で決まります（`app.js` の `RARITY`、レア度1〜10）。
必要クリスタルはレア度帯ごとに 10〜80💎。低レアタグ持ちは救済として費用が少し安くなります。
スカウトビーコン所持中は挑戦時に1個消費し、成功率を +10% します。
スカウト成功 / 失敗 / スルーのそれぞれで、候補キャラの **一言（ボイス）** が表示されます。

### 図鑑（60体）の登録ルール

- **発見済み** … 戦闘勝利後に出会った時点で登録（スカウトの成否・スルーに関係なく残る）
- **加入済み** … スカウト成功でパーティ加入したときに追加記録
- 図鑑セルには発見済みキャラの名前、レア度、タグ、状態、発見回数、スカウト成功回数を表示
- 未発見キャラは「？？？」表示で、レア度とタグは伏せる
- 保存データには発見回数、スカウト成功/失敗回数、スルー回数、別れた回数、初遭遇星、最後に出会った星、最後にパーティにいた星、クリアメンバー回数を `dexStats` として保持
- パーティ4人満員時の入れ替えでは、外れた仲間の `partedCount` を増やす

## セーブ

`localStorage`（キー: `ginga-haguredan-save-v1`）に自動保存。
保存対象は所持クリスタル、クリア済み星、パーティ、発見済み/加入済み図鑑、図鑑統計、ショップアイテム所持数。
古い保存データは読み込み時に安全に移行され、既存の発見済みキャラは発見回数1、加入済みキャラはスカウト成功回数1として初期化されます。
タイトルの「データ消去」または DevTools コンソールで `GAME.reset()`。

## デバッグ用フック

開発中は DevTools コンソールから操作可能:

```js
GAME.save              // 現在のセーブ
GAME.goShooting(0)     // 0番目の星の航行フェーズへ
GAME.goBattle(0)       // 0番目の星の戦闘へ
GAME.winShooting()     // 航行を即クリア（到達）扱いに
GAME.completeMission() // 現在のミッションを成功扱いに
GAME.setBonus("enemyHp" | "allyHp" | "scoutRate" | "crystal") // 航行ボーナスを手動セット
GAME.bonus             // 現在の航行ボーナス
GAME.characterCatalog  // 正式な仲間60人カタログ
GAME.bossUltimates     // ボス必殺技カタログ
GAME.sh                // 航行フェーズの内部状態（ship 等）
GAME.synergy()         // 現在パーティのタグ集計と発動中シナジー
GAME.constants         // TOTAL_CHARACTERS / FINAL_BOSS_UNLOCK_COUNT など
GAME.dexStats()        // 図鑑統計データ
GAME.addDexStat("c1", "foundCount", 1) // 図鑑統計を手動加算
GAME.discoverHeroes(n) // No.1〜n（最大60）を発見済みにする
GAME.reset()           // セーブ初期化
```

## v0.9 アップデート（戦闘の深化・進行バランス・スカウト改善）

### 戦闘中のスキル選択（キャラ→スキル）
- 「スキル」コマンド → **使うキャラを選ぶ** → そのキャラの**スキル一覧から選んで使用** → 敵ターン。
- 各仲間は `skills`（シグネチャ＋レア度で増加、最大4）を持つ。`たたかう` は全員の通常攻撃。
- スキルは8種（単体/全体/回復/防御UP/敵攻撃DOWN/会心/次強化/低確率大ダメージ）。

### 弱点・耐性（属性相性）
- 攻撃スキルに属性（物理/炎/光/闇）。敵は系統ごとに **弱点×1.5 / 耐性×0.6**。
  - ロボAI系=弱点:炎 耐性:物理 ／ 悪人系=弱点:光 耐性:闇 ／ 宇宙生物系=弱点:闇 耐性:炎 ／ 概念系=弱点:物理 耐性:光
- 敵情報に「弱点：◯／耐性：◯」、ヒット時に「弱点！」「耐性…」を表示。

### 進行バランス
- 敵は2ステージごとに敵Lv+1の感覚で段階強化（HP/攻撃は曲線的に上昇）。レベル表示は控えめ。
- ステージは1勝でクリア。過去ステージ再挑戦OK。ワールドマップに**推奨戦力・危険度★**を表示。

### スカウト：4人満員時の入れ替え画面
- 満員でスカウト成功すると選択画面：**誰と別れて入れる / 加入せず図鑑だけ登録**。
- 別れた仲間は消えず**図鑑に残り、`dexStats.partedCount`（別れた回数）が増える**。
- 「図鑑だけ」はクリスタル消費なし・パーティ不変。

### クリスタル経済（さらに貴重に）
- 通常 **+1** / 大 **+6〜8** / 回復はかなり低確率。ミッション成功ボーナス **+10（強化+15）**。
- 出現率：通常16% / 大3% / 回復2%。体感は1航行あたり概ね 5〜15💎（うまくいけば20〜30、失敗0〜5）。
